# Domain Shift in Limit Order Book Forecasting

This repository organizes research notebooks for a study of **domain shift in limit-order-book (LOB) data** and its consequences for mid-price forecasting. The project covers three interconnected threads:

1. **The forecasting task** — characterizing mid-price prediction in the LOB setting.
2. **Measuring distribution shift** — quantifying how statistical properties of LOB data change across markets, using the FI-2010 dataset as the in-distribution reference and LOBSTER (US equities) as the out-of-distribution target.
3. **Adaptation strategies** — exploring how models can be adapted to distribution shift via finetuning, foundation models, and finetuning on synthetic data that is closer to the target distribution than the train.

---

## Datasets

Three datasets are used across the notebooks:

| Dataset | Description | Size | Normalization |
|---|---|---|---|
| **LOBSTER** | US NASDAQ equity order book data (AAPL, MSFT; 2012-06-21). 10-level snapshots + message stream. Sourced from `totalorganfailure/lobster-data` on HuggingFace. | AAPL: 400,391 events; MSFT: 668,765 events | Raw prices (integer hundredths of cents) |
| **FI-2010** | Finnish NASDAQ Nordic equity benchmark. 149-feature matrix: 40 LOB features (10 levels × 4), 103 handcrafted microstructure features, and 5 label horizons. Sourced from the DeepLOB GitHub repository. | 254,750 timestamps | Z-score normalized |
| **LOBS5 / LOB-Bench** | Synthetic LOB data for GOOG, generated by the LOB-Bench simulator. Delivered as raw message streams that must be reconstructed into L2 snapshots. Sourced from `peernagy/lob_bench` on HuggingFace. | ~72,600 events (up to 1,500 message files) | Raw prices |

## Notebooks

### 1. `LOB_Data_Exploration_with_LOBS5.ipynb`

**Exploratory analysis of LOB distributions across markets and vs. synthetic data.**

This notebook establishes the empirical foundation of the project. It loads all three datasets, engineers a shared feature set, and produces a rich set of visualizations to build intuition for where and how distributions diverge.

#### Feature Engineering

A common schema is computed for each dataset:

- `mid_price`, `spread` — best ask/bid midpoint and gap
- `mid_return` — tick-by-tick percent change of mid-price
- `order_imbalance` — `(bid_vol_1 − ask_vol_1) / (bid_vol_1 + ask_vol_1)`
- `total_bid_depth`, `total_ask_depth` — aggregate volume across 10 levels
- `inter_arrival_time` — time between consecutive snapshots
- `labels_h{10,20,50}` — multi-horizon classification targets (Down / Stationary / Up)

#### Preprocessing Challenges

**LOBS5** requires reconstructing L2 order book snapshots from scratch by replaying a raw message stream (handling order submissions, cancellations, and executions). Features are computed *within* each message file segment to avoid spurious returns at file boundaries, with a `segment_id` column tracking provenance.

**FI-2010** is stored as a transposed feature matrix; the notebook extracts the first 40 rows (the LOB price/volume matrix) and maps them to the LOBSTER-compatible schema.

#### Visualizations

Three multi-panel figures are produced:

- **Cross-asset** (`cross_asset_domain_shift.png`): AAPL vs. MSFT — standardized return KDE, spread histogram, order imbalance KDE, tail CCDF (log scale vs. Gaussian reference), depth profile by level, label distribution at h=50.
- **Cross-market** (`cross_market_domain_shift.png`): FI-2010 vs. LOBSTER-AAPL — return KDE, tail CCDF, imbalance KDE, QQ plot (3,000-sample), depth profiles, label distributions for all three datasets.
- **Synthetic vs. real** (`synthetic_domain_shift.png`): LOBS5 vs. LOBSTER and FI-2010 — 6-panel comparison of standardized distributions with KS / Wasserstein summary statistics.

A `dataset_summary_statistics.csv` table is also exported with descriptive statistics for each dataset (spread, return kurtosis, depth, etc.).

### 2. `LOB_Domain_Shift_Quantification.ipynb`

**Rigorous, metric-driven quantification of distributional divergence across all three dataset pairs.**

Where the first notebook builds visual intuition, this notebook makes the shift precise. It computes a battery of complementary divergence metrics for five core microstructure features across every dataset pair, exporting slide-ready figures and a structured metrics table.

#### Research Questions

- How large is the distributional gap between FI-2010 and LOBSTER — the in-distribution and out-of-distribution split used for forecasting experiments?
- How closely does LOBS5 synthetic data resemble real LOB data, and which features drive the largest discrepancy?
- Do different divergence measures tell a consistent story, or do they highlight different aspects of the shift?

#### Metric Suite

For each of the five features (`mid_return`, `spread`, `order_imbalance`, `total_bid_depth`, `total_ask_depth`) and each of the three dataset pairs, the notebook computes:

- **Kolmogorov-Smirnov test** — non-parametric test statistic and p-value
- **KL Divergence & Jensen-Shannon Divergence** — computed from 80-bin histograms; JS is symmetric and bounded, making it easier to compare across pairs
- **Wasserstein-1 distance** — Earth Mover's Distance on sorted samples; interpretable as the minimum transport cost between distributions
- **Moment gaps** — differences in mean, standard deviation, skewness, and excess kurtosis between the two datasets

Before any metric is computed, each feature pair is standardized using the pooled median and IQR (robust to outliers), which puts the scale-heterogeneous datasets on a common footing. Each feature is then subsampled to 80,000 points (random seed 42) for consistency.

#### Key Findings

All 15 pairings (3 pairs × 5 features) are statistically significant (p ≪ 0.001). The nature and magnitude of shift varies considerably by feature:

- **Spread and depth** show near-complete separation (KS ≈ 1.0, JS ≈ 0.58–0.68) between FI-2010 and the other two datasets, driven largely by FI-2010's Z-score normalization collapsing the raw price scale.
- **Mid-return and order imbalance** exhibit more moderate shift (KS 0.07–0.16), reflecting genuine microstructural differences in price dynamics and liquidity balance across markets.
- **LOBS5 vs. LOBSTER-AAPL** shows milder divergence overall (KS 0.07 for returns, 0.16 for imbalance), suggesting the synthetic generator captures some real-market dynamics but still diverges on spread and depth.

#### Outputs

| File | Contents |
|---|---|
| `cross_market_domain_shift.png` | 6-panel figure: FI-2010 vs. LOBSTER-AAPL (return KDE, tail CCDF, imbalance KDE, QQ plot, depth profiles, label distributions) |
| `synthetic_vs_real_domain_shift.png` | 6-panel figure: LOBS5 vs. real data (distributions of all features + mean Wasserstein bar chart) |
| `dataset_summary_statistics.csv` | Per-dataset descriptive statistics |
| `domain_shift_metrics.csv` | Full metrics table: 15 rows × 13 columns (KS, KL, JS, Wasserstein, moment gaps) |

### 3. `chronos2_lob_benchmark.ipynb`

**Chronos-2 as a universal transfer benchmark for limit order book mid-price forecasting.**

This notebook evaluates `Chronos-2` (≈120M params) as a time-series foundation model for cross-market transfer on LOB data. It compares three training conditions — zero-shot (pretrained only), fine-tuned on FI-2010, and fine-tuned on FI-2010 + LOBS5 synthetic — and evaluates transfer performance on LOBSTER (AAPL by default).

#### Experimental Setup

- **Context window:** T = 100 LOB snapshots (DeepLOB-style inputs)
- **Prediction horizons:** k ∈ {10, 20, 50}
- **Task:** 3-class mid-price movement classification (Down / Stationary / Up) derived from k-step median forecasts
- **Metrics:** Macro-F1, per-class F1, accuracy (publication-ready tables and per-horizon summaries)
- **Implementation:** uses `chronos-forecasting` Chronos2Pipeline + `Chronos2Dataset` / `Chronos2Trainer`; supports CPU / MPS / CUDA device maps

#### Fine-tuning & evaluation notes

- Fine-tuning on FI-2010 (and FI-2010 + LOBS5) is performed with overlapping window segments and configurable context/prediction lengths. Training arguments and hyperparameters are set in the notebook (learning rate, batch size, epochs, checkpoint directory).
- Evaluation processes windows in batches with a fixed stride and reports aggregated metrics across tickers and horizons.

#### Outputs

- `chronos2_f1_by_horizon.png` — Macro-F1 by horizon and training condition
- `chronos2_per_class_f1.png` — Per-class F1 breakdown averaged across conditions
- `chronos2_confusion_matrices.png` — Normalized confusion matrices (best condition per horizon)
- `chronos2_results.csv` — Full results table (Condition, Ticker, k, per-class F1, accuracy, sample counts)
- `chronos2_checkpoints/` — directory used to store fine-tuned model checkpoints

These artifacts can be regenerated by running `chronos2_lob_benchmark.ipynb` end-to-end; see the notebook for required packages and exact commands.
